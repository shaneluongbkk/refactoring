Inlining Methods

Last updated: 11/17/2022

âœ« Incline Method

Problem

When a method body is more obvious than the method itself. 


   class Discount {
      // do something
      int getDiscountPercent() {
         return moreThanTwoSimilarItemsBought() ? 30 : 5;
      }
      boolean moreThanTwoSimilarItemsBought() {
         return numberOfSimilarItemsBought > 2;
      }
   }

Solution

Put this code in a distinct new method (or function) and call the new method instead of the previous code.


   class Discount {
      // do something
      int getDiscountPercent() {
         return numberOfSimilarItemsBought > 2 ? 30 : 5;
      }
   }

Why refactor?

A method only transfers control to another method. This delegation poses no issues in and of itself. However, when there are numerous such ways, they combine to form a jumble that is challenging to sort through.


âœ« Extract Variable

Problem

You have a difficult-to-understand expression. 


   void renderBanner() {
      if ((platform.toUpperCase().indexOf("MAC") > -1) &&
            (browser.toUpperCase().indexOf("IE") > -1) &&
            wasInitialized() && resize > 0 )
      {
         // do something
      }
   }

Solution

Place the outcome of the expression or its parts in self-explanatory variables.


   void renderBanner() {
      final boolean isMacOs = platform.toUpperCase().indexOf("MAC") > -1;
      final boolean isIE = browser.toUpperCase().indexOf("IE") > -1;
      final boolean wasResized = resize > 0;
      
      if (isMacOs && isIE && wasInitialized() && wasResized) {
         // do something
      }
   }

Why refactor?

The fundamental rationale for extracting variables is to simplify a complex expression by separating it into intermediate sections.


âœ« Extract Class

Problem

When one class does the work of two, awkwardness results. 


   void student() {
      studentID;
      gradeSemester1;
      gradeSemester2;
      gradeSemester3;
      gradeSemester4;

      //functions
      getGradeBook();
      gpaCalculation();
      saveGradebyID();
   }

Solution

Instead, make a new class and include the properties and methods that are responsible for the required functionality.


   void student() {
      studentID;
      studentGradeBook();
   }

   void studentGradeBook() {
      gradeSemester1;
      gradeSemester2;
      gradeSemester3;
      gradeSemester4;

      //functions
      getGradeBook();
      gpaCalculation();
      saveGradebyID();
   }

Why refactor?

Classes are always clear and simple to understand at the start. They do their job and keep to themselves, so to speak, without interfering with the work of other classes. However, as the software grows, a method, then a field... and soon, certain classes are executing more tasks than anyone could have imagined.
